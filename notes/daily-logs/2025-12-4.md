1. **Database introduction**

   - So databases are used to store data in a structured way, and there are two main types of databases that you should be aware of from an AWS perspective:

     . Relational database
     . Non-Relational database

2. **AMAZON RDS Overview**

   - So RDS is a managed relational database service that allows you to create relational databases in the cloud. So it supports multiple database engines, including MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, and Amazon's own database engine called Aurora.

   - RDS automates time-consuming tasks such as database provisioning, patching, backup, recovery, and scaling, allowing you to focus on your applications rather than database administration.

3. **AMAZON AURORA Overview**

   - Aurora - is a proprietary technology from AWS and is cloud-optimized.

   - So Amazon Aurora is a relational database engine that is fully managed by AWS and is compatible with MySQL and PostgreSQL. Aurora is designed to provide high performance, scalability, and availability for database workloads.

   - Aurora automatically handles tasks such as backups, software patching, monitoring, and scaling, allowing you to focus on your applications rather than database administration.

   - Aurora also offers features such as read replicas for improved read performance, multi-AZ deployments for high availability, and serverless configurations for automatic scaling based on demand.

   - - So, from an exam perspective, RDS and Aurora are going to be the two ways for you to create relational databases on AWS. They're both managed and Aurora is going to be more cloud-native, whereas RDS is going to be running the technologies you know directly as a managed service.
