2. **IAM POLICIES STRUCTURE**

   - Consists of -
    . Version: policy language version, always include number, so usually it's 2012-10-17.
    . Id: an identifier for the policy (optional).
    . Statement: one or more individual statements (required).

   - Statements consists of  -
    . Sid: an identifier for the statement (optional).
    . Effect: whether the the statement allow or denied access (Allow,Deny).  
    . Principal: account/user/role to which policy applied to
    . Action: is the list of API calls that will be either denied or allowed based on the effect.
    . Resources:  is a list of resources, to which the actions will be applied to.
    . Condition: condition for when this policy is in effect (optional).

4. **IAM USER PASSWORD POLICY**

   - Strong Passwords- Higher Security for the Account.
   - In AWS you can setup a password policy:
    . Set a minimum password length
    . Require specific character types:
      . Including uppercase letters
      . Lowercase
      . Numbers
      . Non-alphanumeric characters
    . Allow all IAM Users to change their passwords
    . Require users to change their password after sometime (password expiration)
    . Prevent password re-use
   - Multi Factor Authentication (MFA)
    . It is compulsory to setup MFA in AWS because users have access to your account and they can possibly do a lot of things, especially if they're administrators. They can change configuration, delete resources and other things.
    . You want to protect your Root accounts and IAM Users by using MFA method.
   - We can edit our password policy in Root account

5. **MFA DEVICES OPTIONS IN AWS**

   - Virtual MFA device:
     . Google Authenticator (phone only).
     . Authy (phone only).
     Support for multiple Token on a single device
   - Universal Second Factor (U2F) Security key
     . YubiKey by Yubico (3rd party) not from AWS.
     So this YubiKey supports multiple root and IAM users using a single security key.
   - Hardware Key Fob MFA Device provided by Gemalto (3rd party).
   - Hardware Key Fob MFA Device for AWS GovCloud (US) provided by SurePassID (3rd party).

6. **HOW CAN USERS ACCESS AWS**

   - To access AWS we have three options:
     . AWS Management Console (Protected by password + MFA).
     . AWS Command Line Interface (CLI): Protected by Access Keys.
     . AWS Software Developer Keys (SDK) - for code: Protected by Access Keys.
   - Access Keys are generated through AWS Console.
   - User manage their own Access Keys.
   - Access are secret just like password don't share them, So really make sure that you treat your access key ID just like your username, and your secret access key just like your password, you do not share them with other people.

7. **WHAT'S THE AWS CLI**

   - A tool that allow you to interact with AWS services using commands in your command-line shell.
   - Direct access to the public APIs of AWS services.
   - You can develop script to manage your resources.
   - It's an open-sourcehttps://github.com/aws/aws-cli.
   - Alternative to use AWS Management Console.

8. **WHAT'S THE AWS SDK**

   - AWS Software Developer kit (AWS SDK).
   - Language-specific APIs (set of libraries ).
   - Enable to access and manage AWS services programmatically.
   - Is not something we use in our terminal is what we embedded within our application.
   - It supports many different programming languages, such as JavaScript, Python, PHP.NET, Ruby, Java, Go, Node.js, C++, all of that's our programming languages. and Mobile SDK (Android, IOS,...).
   -IoT Devices SDK: (Embedded C, Arduino,...) and the IoT, so Internet of Things device SDK in case you're using some thermal sensors or bike locks that are connected, all these kinds of things.
   - Example: AWS CLI is built on AWS SDK for Python.
